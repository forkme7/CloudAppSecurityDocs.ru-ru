# <a name="protecting-your-organization-against-threats"></a>Защита организации от угроз  

Ищите представляющие большой риск проблемы с использованием и с безопасностью в облаке, выявляйте аномальное поведение пользователей и предотвращайте угрозы в подключенных облачных приложениях. Повышайте видимость действий пользователей и администраторов и определяйте политики для автоматического оповещения о подозрительном поведении или конкретных действиях в санкционированных окружениях, которые вы считаете подозрительными. Используйте накопленные корпорацией Майкрософт сведения об анализе угроз и исследовании безопасности. Политики обнаружения угроз помогают гарантировать, что для ваших санкционированных приложений используются все необходимые средства защиты безопасности и что вы сохраняете контроль над ними.
 
## <a name="mass-download-by-a-single-user-data-exfiltration-by-an-insider"></a>Массовое скачивание одним пользователем (инсайдерская эксфильтрация данных)

Этот вариант использования распространяется на Office 365, G Suite, Box, Dropbox, Salesforce.

### <a name="the-threat"></a>УГРОЗА
Злоумышленник с доступом к системе может вести эксфильтрацию данных из Office 365 или других облачных приложений, скачивая множество файлов или обращаясь к множеству файлов за короткий период.

### <a name="the-solution"></a>РЕШЕНИЕ
Необходимо обнаружить, когда пользователь скачивает большое количество файлов или обращается к большому количеству файлов за короткое время.

#### <a name="prerequisites"></a>Предварительные условия

[Подключите](enable-instant-visibility-protection-and-governance-actions-for-your-apps.md) хотя бы одно облачное приложение к Cloud App Security.

#### <a name="setting-up-monitoring"></a>Настройка мониторинга

1.  По умолчанию Cloud App Security сканирует сеть, чтобы получить базовые показатели использования. На их основе Cloud App Security определяет, что ваши пользователи обычно делают в облаке и когда они это делают. 

2. Настройте политику, которая будет следить за скачиванием большого объема данных из облачных приложений и оповестит вас о том, что произошло что-то необычное.

    1. На странице **Политики** выберите команду [**Создать политику действий**](user-activity-policies.md). 
   

    2. В поле [**Шаблон политики**](policy-template-reference.md) выберите пункт **Массовая загрузка одним пользователем**.
    
    3. При необходимости можно использовать тонкую настройку с помощью фильтра повторяющегося действия.
    
    4. Нажмите кнопку **Create** (Создать). 
   
     
2. Анализ совпадений
    
    1. На странице **Политики** щелкните имя политики, чтобы перейти к **отчету политики** и просмотреть совпадения, инициируемые для политики.

    2. Для анализа совпадения щелкните его, чтобы открыть панель действий. На панели действий вы можете увидеть другие политики, которым соответствует это действие. 
    
    3. Вы можете проверить файлы, скачанные пользователем, щелкнув объекты действий на **панели действий** и выбрав имя файла. При этом откроется файл из соответствующей таблицы. В этом представлении можно просмотреть, принадлежат ли файлы пользователю, кто осуществляет к ним общий доступ, и предназначены ли они для повседневной работы или являются внутренней интеллектуальной собственностью, которую не следует скачивать.
     


#### <a name="validating-your-policy"></a>Проверка политики

1. Для имитации оповещения скачайте из подключенных облачных приложений несколько документов, число которых превышает установленное пороговое значение, в течение короткого периода с учетом настроенного в политике временного интервала (например, 30 скачиваний менее чем за 5 минут).
3. Перейдите к отчету политики. В нем должно появиться оповещение о соответствии для политики действий. 
4. Щелкнув совпадение, вы увидите, какие файлы были скачаны.  

#### <a name="removing-the-risk"></a>Устранение риска

После проверки и настройки политики удалите возможные ложные срабатывания, найденные политикой, например, учетные записи службы, синхронизирующие файлы, папки с фотографиями с корпоративных мероприятий и т. д. Затем сделайте следующее. 
  1. Вы можете предпринять [действия системы управления](governance-actions.md), открыв активность в панели действий и выбрав файл в области **Объекты-действия**.

  2. Узнайте у пользователей, зачем им на компьютере нужны копии стольких корпоративных файлов.

 
## <a name="admin-activity-from-outside-your-organizations-network"></a>Активность администратора извне сети организации

Этот вариант использования распространяется на Office 365, G Suite, Box, Dropbox, Salesforce и Okta.

### <a name="the-threat"></a>УГРОЗА
Учетная запись администратора была скомпрометирована внешним злоумышленником. Учетные записи администраторов являются наиболее важными в организации, так как имеют наибольший объем прав и доступ ко всем данным организации. Обычно администратор работает из офиса и находится в рамках служебных обязанностей, а не действует из удаленных расположений или с неопознанных устройств.

### <a name="the-solution"></a>РЕШЕНИЕ
Определяйте ситуации, когда пользователи, выдающие себя за администраторов, подключаются к облачному приложению с внешних IP-адресов и выполняют любые операции по администрированию.

#### <a name="prerequisites"></a>Предварительные условия

- [Подключите](enable-instant-visibility-protection-and-governance-actions-for-your-apps.md) хотя бы одно облачное приложение к Cloud App Security.

- Последовательно выберите пункты **Параметры** > **Диапазоны IP-адресов** и добавьте диапазоны IP-адресов для внутренних подсетей и их общедоступных IP-адресов исходящего трафика, а затем пометьте их как **Корпоративные**.

#### <a name="setting-up-monitoring"></a>Настройка мониторинга

1.  Начните наблюдение за облачными приложениями, настроив политику, которая будет следить за внешними операциями администрирования в облачных приложениях и оповестит вас о том, что произошло что-то необычное.

    1. На странице **Политики** выберите команду [**Создать политику действий**](user-activity-policies.md). 
   

    2. В поле [**Шаблон политики**](policy-template-reference.md) выберите **Действие администрирования с некорпоративного IP-адреса**, задайте для параметра **Тип действия** значение **Вход** и выберите **Пользователь**, **Из группы**, а затем выберите группу, к которой относятся ваши администраторы.
    
    3. Можно настроить политики, задав число повторяющихся действий, которые следует рассматривать как подозрительные.
    
    4. Нажмите кнопку **Create** (Создать). 
   
     
2. Анализ совпадений
    
    1. На странице **Политики** щелкните имя политики, чтобы перейти к **отчету политики** и просмотреть совпадения, инициируемые для политики.

    2. Для анализа совпадения щелкните его, чтобы открыть панель действий. На панели действий вы можете увидеть другие политики, которым соответствует это действие. 
     
#### <a name="validating-your-policy"></a>Проверка политики

1. Для имитации оповещение на компьютере с IP-адресом, находящимся вне вашей корпоративной сети, выполните несколько операций администрирования, число которых превышает установленное пороговое значение, в течение короткого периода с учетом настроенного в политике временного интервала (например, 1 действие менее чем за 5 минут).
3. Перейдите к отчету политики. В нем должно появиться оповещение о соответствии для политики действий. 
4. Щелкнув совпадение, вы увидите, какие действия были выполнены. 

#### <a name="removing-the-risk"></a>Устранение риска

После проверки и настройки политики удалите возможные ложные срабатывания, найденные политикой. Затем сделайте следующее. 
  1. Вы можете предпринять [действия системы управления](governance-actions.md), открыв активность в панели действий и выбрав имя в области **Пользователь**.

  2. На открывшейся странице пользователя можно просмотреть граф действий пользователя за последний месяц, расположения, с которыми пользователь работал в течение последнего месяца, а также членство в группах, активность в приложениях и учетные записи. 
  
  3. При необходимости можно выбрать элемент **контакт**, чтобы оповестить пользователя по электронной почте о нарушении в его учетной записи.

 ![Внешнее автоматическое управление](./media/auto-gov-external.png)

   2. После полной проверки вы можете настроить для политики выполнение действий автоматического управления. Например, вы можете **Приостановить пользователя** или **Remove user's collaborations** (Удалить совместную работу пользователя).
