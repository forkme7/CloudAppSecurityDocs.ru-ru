# <a name="daily-activities-to-protect-your-cloud-environment"></a>Ежедневные мероприятия для защиты облачной среды
После запуска Cloud App Security, настройки потоков данных, санкционирования приложений, к которым сотрудники должны иметь доступ, и настройки политик для отслеживания облачной среды можно воспользоваться Cloud App Security для контроля и защиты облака, а также для управления рисками.  
  
Эта статья описывает, что именно следует делать на ежедневной основе.  
  
## <a name="check-the-dashboard"></a>Проверка панели мониторинга  
При открытии портала Cloud App Security выводится обзорное представление с открытыми оповещениями, нарушениями действий, нарушениями содержимого, картой действий, которая указывает географическое расположение для выполненного пользователем действия, а также тенденциями использования подключенных приложений в облачной среде.  
  
Рекомендуется ежедневно проверять эту панель мониторинга, чтобы узнавать о новых оповещениях и обрабатывать их. В этом месте также удобно следить за работоспособностью облачной среды, чтобы получить общее представление о том, что в ней происходит.  
  
![панель мониторинга](./media/dashboard.png "dashboard")  
  
## <a name="handle-your-alerts"></a>Обработка оповещений  
Оповещения являются ключом к более глубокому понимаю облачной среды. На основе полученных результатов можно создавать новые политики. Например, вы можете заметить, что администратор входит в систему из Гренландии, и решить создать политику, которая автоматически приостанавливает действие учетной записи администратора при входе из Гренландии.  
  
Рекомендуется просматривать все оповещения и руководствоваться ими при изменении политик. Если безобидные события расцениваются как нарушение политик, следует уточнить политики, чтобы уменьшить число ненужных предупреждений.  
  
-   В разделе **Открытые оповещения** щелкните **Просмотреть все оповещения**.  
  
     Это дает полное представление о любой подозрительной активности или нарушении заданных политик и помогает дополнительно обезопасить механизмы защиты облачной среды.  
  
     ![предупреждения](./media/alerts.png "alerts")  
  
-   Для каждого оповещения следует провести расследование и определить характер нарушения и необходимые меры.  
  
     Можно отфильтровать оповещения по параметрам **Тип оповещений** или **Серьезность**, чтобы сначала обрабатывать наиболее важные из них.  
  
     Щелкните определенное оповещение. В зависимости от его типа вам будут предложены различные действия, которые можно выполнить до разрешения оповещения.  
  
     Существует три типа нарушений, с которыми приходится сталкиваться при анализе оповещений:  
  
    -   **Серьезные нарушения**, которые требуют незамедлительного реагирования.  
  
         Примеры  
  
         В случае оповещения о подозрительной активности рекомендуется приостановить действие учетной записи, пока пользователь не сменит пароль.  
  
         В случае утечки данных можно ограничить разрешения или поместить файл в карантин.  
  
         Если обнаружена новая несанкционированная служба, можно заблокировать доступ к ней на прокси-сервере или в брандмауэре.  
  
    -   **Сомнительные нарушения**, при которых требуется дополнительный анализ.  
  
         Вы можете обратиться к пользователю или его руководителю, чтобы выяснить характер активности.  
  
         Оставьте вопрос о такой активности открытым, пока не соберете дополнительные сведения.  
  
    -   **Авторизованные нарушения или аномальное поведение**, вызванное правомерным использованием.  
  
         Отмените такое оповещение.  
  
-   После завершения этого процесса пометьте оповещение как разрешенное.

## <a name="use-policies-to-assess-risk"></a>Использование политик для оценки рисков  
После просмотра открытых оповещений перейдите в **Центр политик** для просмотра нарушений политики, которые не привели к выводу оповещений.  
  
-   На портале Cloud App Security щелкните **Управление** и выберите пункт **Политики**.  
  
-   Выберите конкретную политику, чтобы просмотреть список совпадений политики **Нарушается сейчас**, которые не вызывают оповещения.  
  
-   Щелкайте нарушения по одному, чтобы решить, что с ними делать. Дополнительные сведения о действиях системы управления см. далее.  
  
     Например, если политика настроена на поиск нарушений соответствия и кто-то сохраняет номера кредитных карт в файлах в OneDrive, в политике возникает совпадение.  
  
     ![совпадения pci](./media/pci-matches.png "pci matches")  
  
-   Щелкните это совпадение, чтобы просмотреть файлы, вызвавшие нарушение политики.  
  
     ![совпадения содержимого pci](./media/pci-content-matches.png "pci content matches")  
  
     Можно щелкнуть сам файл для получения сведений о нем.  
  
     Можно щелкнуть элемент **Участники совместной работы**, чтобы узнать, кто именно имеет доступ к этому файлу.  
  
     Можно щелкнуть **Соответствия** для просмотра номеров кредитных карт.  
  
     ![совпадения содержимого ccn](./media/content-matches-ccn.png "content matches ccn")
