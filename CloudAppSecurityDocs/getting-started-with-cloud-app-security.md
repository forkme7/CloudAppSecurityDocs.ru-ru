# <a name="getting-started-with-cloud-app-security"></a>Приступая к работе с Cloud App Security
Cloud App Security помогает клиентам использовать преимущества облачных приложений, а также осуществлять управление за счет улучшенного контроля активности и повышенной защиты важных данных компании.  В этой документации описаны приведенные ниже шаги, помогающие начать работу с Cloud App Security.  
  
Для использования продукта ваша организация должна иметь лицензию на Cloud App Security. Дополнительные сведения см. в статье [How to buy Cloud App Security](https://www.microsoft.com/server-cloud/products/cloud-app-security/default.aspx) (Приобретение Cloud App Security), а также ознакомьтесь со статьей [Licensing resources](https://www.microsoft.com/server-cloud/products/cloud-app-security/default.aspx) (Ресурсы по лицензированию).  
  
>[!NOTE]
>Лицензия Office 365 не требуется для Cloud App Security.  
  
## <a name="get-started-quickly-with-cloud-app-security"></a>Быстрое начало работы с Cloud App Security  
  
|Необходимые действия:|Задача|Требуется?|Как...|Описание|  
|-------------------------|----------|---------------|-------------|-----------------|  
|ШАГ 1. [Настройка Cloud Discovery](set-up-cloud-discovery.md).|Отправка журналов трафика|Обязательный|**Создание непрерывного отчета Cloud Discovery**<br /><br /> 1.   Выберите **Параметры** -> **Параметры Cloud Discovery**.<br /><br /> 2.    Щелкните **Upload log automatically** (Автоматическая отправка журнала).<br /><br /> 3.   Добавьте источники данных на вкладке **Источники данных**.<br /><br /> 4.    Настройте сборщик журналируемых данных на вкладке **Сборщики данных журнала**.<br /><br /> Создание отчета о снимке Cloud Discovery<br /><br /> 1.    Выберите **Обнаружение** -> **Создать отчет о снимке** и выполните действия.|**Зачем настраивать отчеты Cloud Discovery?** Получение представления о теневых ИТ-ресурсах в организации имеет наиважнейшее значение.  <br />После анализа журналов можно легко определить, какие облачные приложения используются, какими людьми и на каких устройствах.|  
|ШАГ 2. [Включение мгновенной видимости, защиты и действий управления для приложений](enable-instant-visibility-protection-and-governance-actions-for-your-apps.md).|Подключение приложений|Обязательный|1.   Выберите **Параметры** -> **Санкционированные приложения**.<br /><br /> 2. Щелкните **Подключить приложение** и выберите приложение.<br /><br /> 3. Выполните шаги настройки для подключения приложения.|**Для чего подключать приложение?**<br /><br /> Только после подключения приложения можно будет получить более четкое представление для анализа действий, файлов и учетных записей в облачной среде для облачных приложений.|  
|ШАГ 3. [Управление облачными приложениями с помощью политик](control-cloud-apps-with-policies.md).|Создание политик|Обязательный|**Создание политик**<br /><br /> 1.  Выберите **Управление** -> **Шаблоны**.<br /><br /> 2.    Выберите из списка шаблон политики и щелкните (+) **Создать политику**.<br /><br /> 3.  Настройте политику под свои потребности (выберите фильтры, действия и другие элементы конфигурации) и нажмите кнопку **Создать**.<br /><br /> 4.    На вкладке **Политики** выберите созданную политику, чтобы просмотреть связанные с ней соответствия (действия, оповещения, файлы и т. д).<br /><br /> Совет. Создайте политику для каждой **категории риска**, чтобы охватить все сценарии безопасности облачной среды.|**Как политики могут помочь вашей организации?**<br /><br /> Политики предоставляют средства для отслеживания тенденций, просмотра угроз безопасности и создания собственных отчетов и оповещений. С помощью политик можно создать различные действия системы управления, средства для общего доступа к файлам и защиты от потери данных.|  
|ШАГ 4. [Персонализация взаимодействия](general-setup.md#Adallom_mailsettings).|Добавление сведений об организации|Рекомендуется|**Указание параметров электронной почты**<br /><br /> 1. Выберите **Параметры** -> **Параметры почты**.<br /><br /> 2.   В разделе **Удостоверение отправителя письма** введите свои адреса электронной почты и отображаемое имя.<br /><br /> 3. В разделе **Дизайн письма** отправьте шаблон сообщения электронной почты для вашей организации.<br /><br /> **Настройка уведомлений администратора**<br /><br /> 1.    Щелкните свое имя пользователя в панели навигации и выберите **Параметры пользователей**.<br /><br /> 2.    В разделе **Уведомления** настройте методы, которые необходимо задать для системных уведомлений.<br /><br /> 3.  Нажмите кнопку **Сохранить**.<br /><br /> **Настройка метрик оценки**<br /><br /> 1.  Выберите **Параметры** -> **Параметры Cloud Discovery**.<br /><br /> 2.    В разделе **Score metric configuration** (Настройка метрик оценки) настройте важность различных значений риска.<br /><br /> 3.    Нажмите кнопку **Сохранить**.<br /><br /> Теперь оценки риска, назначенные обнаруженным приложениям, настраиваются точно в соответствии с потребностями и приоритетами организации.|**Для чего нужна персонализация среды?**<br /><br /> Некоторые функции работают оптимально, когда они настроены в соответствии с вашими потребностями.  <br />Улучшите взаимодействие с пользователями за счет собственных шаблонов сообщений электронной почты, выберите получаемые уведомления и настройте метрику оценки риска в соответствии требованиями вашей организации.|  
|ШАГ 5. [Организация данных в соответствии с потребностями](general-setup.md#IPtagsandRanges).|Настройка важных параметров|Рекомендуется|**Создание тегов IP-адресов**<br /><br /> 1.   Выберите **Параметры ** -> **Теги IP-адреса**.<br /><br /> 2. Щелкните (+) **Добавить диапазон IP-адресов**.<br /><br /> 3.    Введите **сведения**, **расположение**, **теги** и **категорию** для диапазона IP-адресов.<br /><br /> 4. Нажмите кнопку **Create** (Создать).<br /><br /> Теперь можно использовать теги IP-адресов при создании политик, фильтрации и при создании представлений данных.<br /><br /> **Создание представлений**<br /><br /> 1.  Выберите **Параметры** -> **Параметры Cloud Discovery**.<br /><br /> 2.    В разделе **Представления данных** щелкните (+) **Добавить представление данных**.<br /><br /> 3.  Выполните шаги настройки.<br /><br /> 4.  Нажмите кнопку **Создать**.<br /><br /> Теперь можно просматривать обнаруженные данные в соответствии со своими предпочтениями, например по подразделениям или диапазонам IP-адресов.<br /><br /> **Добавление доменов**<br /><br /> 1.    Выберите **Параметры** -> **Общие параметры**.<br /><br /> 2.    В разделе **Сведения об организации** добавьте внутренние домены организации.<br /><br /> 3. Нажмите кнопку **Сохранить**.|**Почему следует настроить эти параметры?**<br /><br /> Эти параметры улучшают и упрощают управление различными функциями в консоли. С помощью тегов IP-адресов проще создавать политики, соответствующие вашим потребностям, точно фильтровать данные и выполнять многие другие задачи. Используйте представления данных для группирования данных в логические категории.|
