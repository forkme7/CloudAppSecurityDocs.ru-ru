# <a name="getting-started-with-cloud-app-security"></a>Приступая к работе с Cloud App Security
Cloud App Security поможет получить преимущества облачных приложений и сохранить контроль над корпоративными ресурсами. Эта служба повышает прозрачность действий в облаке и помогает повысить безопасность корпоративных данных. В этом разделе представлено пошаговое руководство для настройки Cloud App Security и работы с ней.  

Для использования Cloud App Security ваша организация должна иметь лицензию. Дополнительные сведения см. в статье [Приобретение Cloud App Security](https://www.microsoft.com/en-us/cloud-platform/cloud-app-security).  

>[!NOTE]
>Для использования Cloud App Security не требуется лицензия Office 365.  

## <a name="get-started-quickly-with-cloud-app-security"></a>Быстрое начало работы с Cloud App Security  

|Необходимые действия|Задача|Требуется?|Инструкции|Описание|  
|-------------------------|----------|---------------|-------------|-----------------|  
|Шаг 1. [Настройка Cloud Discovery](set-up-cloud-discovery.md).|Отправка журналов трафика|Обязательный|**Создание непрерывного отчета Cloud Discovery**<br /><br /> 1. Выберите **Параметры** > **Параметры Cloud Discovery**.<br /><br /> 2. Выберите пункт **Автоматическая отправка журнала**.<br /><br /> 3. На вкладке **Источники данных** добавьте свои источники.<br /><br /> 4. На вкладке **Сборщики журналов** настройте сборщик журналов.<br /><br /> **Создание отчета о моментальном снимке Cloud Discovery**<br /><br /> 1. Выберите пункты **Обнаружение** > **Создать отчет о моментальном снимке** и выполните указанные действия.|**Зачем настраивать отчеты Cloud Discovery?**<br /> Представление о теневых ИТ-ресурсах в организации имеет наиважнейшее значение. <br />После анализа журналов можно легко определить, какие облачные приложения используются, какими людьми и на каких устройствах.|
|Шаг 2. [Настройка мгновенной видимости, защиты и действий управления для приложений](enable-instant-visibility-protection-and-governance-actions-for-your-apps.md).|Подключение приложений|Обязательный|1. Выберите **Параметры** > **Санкционированные приложения**.<br /><br /> 2. Щелкните пункт **Подключить приложение** и выберите приложение.<br /><br /> 3. Выполните шаги настройки для подключения приложения.|**Для чего подключать приложение?**<br />После подключения приложения можно повысить прозрачность и исследовать действия, файлы и учетные записи для приложений в облачной среде.|
|Шаг 3. [Управление облачными приложениями с помощью политик](control-cloud-apps-with-policies.md).|Создание политик|Обязательный|**Создание политик**<br /><br /> 1. Выберите **Управление** > **Шаблоны**.<br /><br /> 2. Выберите в списке шаблон политики и щелкните пункт (+) **Создать политику**.<br /><br /> 3. Настройте политику (выберите фильтры, действия и другие параметры), а затем выберите команду **Создать**.<br /><br /> 4. На вкладке **Политики** выберите политику, чтобы просмотреть связанные с ней соответствия (действия, файлы, оповещения).<br /><br /> Совет. Создайте политику для каждой **категории риска**, чтобы охватить все сценарии безопасности облачной среды.|**Как политики могут помочь вашей организации?**<br />Вы можете использовать политики для отслеживания тенденций, просмотра угроз безопасности и создания собственных отчетов и оповещений. С помощью политик можно создать действия системы управления, а также задать средства для защиты от потери данных и общего доступа к файлам.|
|Шаг 4. [Персонализация взаимодействия](general-setup.md#Adallom_mailsettings).|Добавление сведений об организации|Рекомендуется|**Указание настроек электронной почты**<br /><br /> 1. Выберите **Параметры** > **Параметры почты**.<br /><br /> 2. В разделе **Удостоверение отправителя письма** введите свои адреса электронной почты и отображаемое имя.<br /><br /> 3. В разделе **Дизайн письма** отправьте в систему шаблон сообщения электронной почты для вашей организации.<br /><br /> **Настройка уведомлений администратора**<br /><br /> 1. На панели навигации выберите свое имя пользователя и перейдите в раздел **Параметры пользователя**.<br /><br /> 2. В разделе **Уведомления** настройте методы, которые необходимо задать для системных уведомлений.<br /><br /> 3. Выберите команду **Сохранить**.<br /><br /> **Настройка метрик оценки**<br /><br /> 1. Выберите **Параметры** > **Параметры Cloud Discovery**.<br /><br /> 2. В разделе **Настройка метрик оценки** настройте важность различных значений риска.<br /><br /> 3. Выберите команду **Сохранить**.<br /><br /> Теперь оценки риска, назначенные обнаруженным приложениям, настраиваются точно в соответствии с потребностями и приоритетами организации.|**Для чего нужна персонализация среды?**<br />Некоторые функции работают оптимально, когда они настроены в соответствии с вашими потребностями. <br />Улучшите взаимодействие с пользователями за счет собственных шаблонов сообщений электронной почты, выберите получаемые уведомления и настройте метрику оценки риска в соответствии требованиями вашей организации.|
|Шаг 5. [Организация данных в соответствии с потребностями](general-setup.md#IPtagsandRanges).|Настройка важных параметров|Рекомендуется|**Создание тегов IP-адресов**<br /><br /> 1. Выберите **Параметры ** > **Теги IP-адреса**.<br /><br /> 2. Выберите пункт (+) **Добавить диапазон IP-адресов**.<br /><br /> 3. Введите **сведения**, **расположение**, **теги** и **категорию** для диапазона IP-адресов.<br /><br /> 4. Выберите команду **Создать**.<br /><br /> Теперь вы можете использовать теги IP-адресов при создании политик, фильтрации и создании представлений данных.<br /><br /> **Создание представлений**<br /><br /> 1. Выберите **Параметры** > **Параметры Cloud Discovery**.<br /><br /> 2. В разделе **Представления данных** щелкните пункт (+) **Добавить представление данных**.<br /><br /> 3. Выполните шаги настройки.<br /><br /> 4. Выберите команду **Создать**.<br /><br /> Теперь можно просматривать обнаруженные данные в соответствии со своими предпочтениями, например по подразделениям или диапазонам IP-адресов.<br /><br /> **Добавление доменов**<br /><br /> 1. Выберите **Параметры** > **Общие параметры**.<br /><br /> 2. В разделе **Сведения об организации** добавьте внутренние домены организации.<br /><br /> 3. Выберите команду **Сохранить**.|**Почему следует настроить эти параметры?**<br />Они помогают лучше управлять функциями в консоли. С помощью тегов IP-адресов проще создавать политики, соответствующие вашим потребностям, точно фильтровать данные и выполнять многие другие задачи. Используйте представления данных для группирования данных в логические категории.|
